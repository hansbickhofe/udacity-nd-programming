When accessing the predefined Class Attributes __doc__ we can get Textinformation that is stored inside the Class

My conclusion:

"Advantages using Classes (OOP)"

Using Classes or OOP Methods is different to conventional, procedural or modular Programming. We combine our Data
(Attributes) and Functions (Methods) to Objects. We are able to process or modify those Objects internaly or let them interact with other Parts of our Program, Classes, Functions or also other Objects.
The Goal is to combine the Data directly with the suitable Code in the Memory for it's processing.
We can create Subclasses that inherit Attributes and Methods from their "Parent" Classes and extend them with additional Data Attributes and/or Methods.
In Phyton for example,all Data is an object
This Principle helps use make more reusable Code and reduces the risk of errors.

Thinking of Classes/Objects is a more abstract View at our Programs. We create Models/Constructors for our Objects and, if needed Hierarchies, to break our work into a smaller and simpler Parts.
Afterwards we can bundle our set of Classes into a Package.

Vocabulary

New Words we learned

Class - An abstract Model that describes Attributes and Methods of Objects
Instance - An Object is an instance of a class
Constructor - A Subroutine that creates the Object
Self - Refers to the Object himself (Self is common Naming convention, we should use)
Instance Variables - A Variable that's unique for each created Object
Instance Methods - A Method that's unique for each created Object
Class Variables - A Variable that's created at Classlevel for all Objects
Inheritance - Creating a class bases on another Class, for reuse of code and extensions.
